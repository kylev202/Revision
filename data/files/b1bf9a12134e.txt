01  Role of the Network Layer 
 
The network layer is Layer 3 of the OSI model. It enables data transfer not only within a single 
network but across multiple interconnected networks — something the Data Link layer (L2) cannot 
do alone. 
 
 
💡  Key Point:  The Data Link layer handles communication within a single broadcast domain (e.g. 
one VLAN). The Network layer extends this across routers to different networks. 
 
Four Core Responsibilities

Responsibility 
What it means 
Addressing 
Assigns unique IP address to each host. Every device in an IP network must

Encapsulation 
Receives Transport layer PDU (segment) → adds L3 header (src/dst IP) →

Decapsulation 
At the destination: strips the L3 header → passes the transport PDU up to the

Packet Routing 
Forwards packets hop-by-hop from source to destination using local Routing

Hop-by-Hop Routing 
In an IP network, routing decisions are made independently at each router (each hop). Every router: 
▸ Inspects the Destination IP address in the packet header. 
▸ Looks up its local routing table to find the best exit interface. 
▸ Forwards the packet to the next hop. 
 
 
 
Hop  — A single transit through one Layer 3 device (router) on the path from source to destination. 
Routing Table  — A table on each router listing known destination networks and their corresponding 
exit interfaces. 
 
⚠️  Important:  The network layer provides device-to-device connectivity only. It does NOT deliver 
data to the correct application — that is the job of the Transport layer and above.

03  IPv4 Characteristics 
IPv4 has three fundamental characteristics that define how it operates:

1. Connectionless 
▸ No handshake or session is established before sending data. 
▸ The sender does not confirm the receiver is reachable before transmitting. 
▸ No acknowledgement is sent when a packet arrives. 
▸ The receiver has no advance notice a packet is coming — it just processes them as they arrive. 
▸ IP is always ready to send and receive; zero overhead for connection setup. 
 
📮  Analogy:  Like posting a letter — you drop it in the post box with no confirmation it was 
delivered, and the recipient doesn't know it's coming. 
 
2. Best-Effort Delivery 
▸ IP does not guarantee packet delivery. 
▸ IP does not guarantee packets arrive in the order they were sent. 
▸ IP does not detect or recover from packet corruption or loss. 
▸ Packets are processed First-In, First-Out (FIFO) — no prioritisation at Layer 3. 
▸ Each packet is routed independently; two packets to the same destination may take different 
paths. 
 
✅  Benefit:  Best-effort means fast processing and low overhead. TCP (Transport layer) handles 
reliability where it is needed. 
 
3. Media Independent 
▸ IPv4 operates identically regardless of the physical medium: copper Ethernet, fibre, serial, 
wireless. 
▸ The Data Link layer adapts the IP packet to the specific medium's frame format. 
▸ However, IP must respect one media constraint: the Maximum Transfer Unit (MTU). 
 
MTU (Maximum Transfer Unit)  — The largest PDU the Data Link layer can carry on a given link. 
Standard Ethernet MTU = 1500 bytes. 
 
When a packet is too large for a link's MTU, it is broken into smaller fragments — this is called IP 
Fragmentation. Each fragment is forwarded and reassembled at the destination. 
Scenario 
MTU Detail 
Ethernet link 
MTU = 1500 bytes (standard) 
Serial link (e.g.) 
MTU may be as low as 200 bytes 
Fragmentation 
If packet (1500B) must cross a 200B MTU link → split into multiple fragments 
Reassembly 
Performed at the final destination host, not intermediate routers

IP Addressing — Structure 
An IPv4 address is a 32-bit number expressed in dotted decimal notation (four octets, each 0–255). It 
follows a hierarchical structure: 
 
▸ Network portion — identifies which network the host belongs to. 
▸ Host portion — identifies the specific host within that network. 
 
🗺️  Example:  192.168.1.10 /24  →  Network: 192.168.1.0  |  Host: .10  |  Broadcast: 192.168.1.255 
 
The subnet mask (or prefix length /n) defines how many bits belong to the network portion. Within 
each network: 
▸ First address (all host bits = 0) → Network address — not assignable to hosts. 
▸ Last address (all host bits = 1) → Broadcast address — not assignable to hosts. 
▸ All addresses in between → valid host addresses. 
 
05  IPv4 Communication Types 
IPv4 supports three types of transmission, each using a different addressing scheme: 
 
 
 
Unicast 
▸ One sender → One receiver. 
▸ Most common type of communication (web browsing, file transfer, SSH, etc.). 
▸ Source and destination IPs are specific host addresses. 
▸ Unicast range: any valid host address not in multicast or broadcast ranges. 
 
Broadcast 
▸ One sender → All hosts in a network.

Two types of broadcast: 
Type 
Details 
Limited Broadcast 
Destination: 255.255.255.255 Stays within the local L2 network. Routers do

Directed Broadcast 
Destination: Network broadcast address (e.g. 192.168.1.255) Originated from

⚠️  Note:  Both types use the universal broadcast MAC address (FF:FF:FF:FF:FF:FF) at Layer 2. 
 
Multicast 
▸ One sender → A selected group of hosts, potentially across multiple networks. 
▸ Hosts must subscribe to a multicast group via a client program (using IGMP). 
▸ Only one copy of the packet is sent until the path branches toward multiple receivers — far 
more efficient than sending individual unicasts to each member.

IPv4 Address Exhaustion & the Future 
IPv4 provides approximately 4.3 billion addresses (2³²). This is no longer sufficient because: 
▸ Every router needs multiple IP addresses (one per interface). 
▸ Every internet-connected device (phones, printers, servers, IoT) requires an address. 
▸ The explosive growth of devices in Asia and globally has accelerated exhaustion. 
▸ NAT provides a workaround but breaks true end-to-end connectivity. 
 
🔮  Solution:  IPv6 uses 128-bit addresses (2¹²⁸ ≈ 340 undecillion addresses) — effectively 
unlimited. Adoption is growing but IPv4 remains dominant in many networks. 
 
07  Address Resolution Protocol (ARP) 
ARP bridges the gap between Layer 3 (IP addresses) and Layer 2 (MAC addresses). Before a 
device can send a frame to another host on the same network, it must know that host's MAC 
address. 
 
Why Two Addresses? 
 
 
In the TCP/IP stack, Layer 2 and Layer 3 use different addressing systems. IP handles end-to-end 
delivery across networks; MAC handles hop-by-hop delivery within each network segment. Every 
time a packet crosses a router, the MAC addresses in the frame change — the IP addresses do not. 
 
🔑  Key Insight:  IP addresses are logical (can change). MAC addresses are physical (tied to 
hardware). ARP is the mechanism that maps one to the other on a local segment. 
 
ARP Operation

Step-by-step ARP process when PC-A (192.168.1.10) wants to reach PC-B (192.168.1.20): 
1. PC-A checks its ARP cache for an entry for 192.168.1.20. 
2. No entry found → PC-A sends an ARP Request as a Layer 2 BROADCAST (ff:ff:ff:ff:ff:ff). 
3. The ARP Request asks: "Who has 192.168.1.20? Tell 192.168.1.10." 
4. All devices on the segment receive the broadcast. Only PC-B recognises its own IP and 
responds. 
5. PC-B sends an ARP Reply as a UNICAST directly back to PC-A with its MAC address. 
6. PC-A stores the mapping (192.168.1.20 → BB:BB:BB:BB:BB:BB) in its ARP cache. 
7. PC-A can now encapsulate IP packets in frames addressed directly to PC-B's MAC. 
 
ARP Message 
Direction  /  Layer 2 Destination 
ARP Request 
Broadcast — sent to ff:ff:ff:ff:ff:ff — all hosts on segment receive it 
ARP Reply 
Unicast — sent only to the MAC of the requesting host 
 
⏱️  ARP Cache:  To avoid repeating this process for every packet, ARP results are stored in an ARP 
cache with a time-to-live. Entries are refreshed if communication continues, or expire if not. 
 
ARP and Routing (Cross-Network) 
When the destination IP is on a different network, PC-A does NOT ARP for the destination host. 
Instead: 
▸ PC-A ARPs for its default gateway (router) MAC address. 
▸ The router receives the packet, decrements TTL, looks up its routing table, and forwards the 
packet. 
▸ At each hop, the sending device ARPs for the next-hop router's MAC — the IP addresses 
remain unchanged throughout. 
 
💡  Remember:  MAC addresses change at every hop (L2 scope). Source and Destination IP 
addresses remain the same end-to-end (L3 scope).

Quick Reference — Key Terms 
 
Term 
Definition 
Layer 3 / Network 
Layer 
OSI layer responsible for logical addressing and routing packets across 
networks. 
IP Packet 
The PDU at Layer 3. Contains L3 header (src/dst IP, TTL, protocol, etc.) + 
payload. 
Router 
Layer 3 device that forwards packets between networks using a routing table. 
Hop 
One transit through a single router on the path from source to destination. 
Routing Table 
Per-router table listing destination networks and corresponding exit

TTL 
Time-to-Live — decremented each hop, preventing infinite routing loops. 
MTU 
Maximum Transfer Unit — largest packet a link can carry (Ethernet: 1500 
bytes). 
Fragmentation 
Splitting a packet into smaller pieces to cross a link with a smaller MTU. 
Connectionless 
No session/handshake before sending; no ACKs at the IP layer. 
Best-Effort 
IP doesn't guarantee delivery, order, or performance. 
ARP 
Address Resolution Protocol — maps IP addresses to MAC addresses on a 
local segment. 
ARP Cache 
Table storing recently learned IP → MAC mappings to avoid repeated ARP 
requests. 
Unicast 
One-to-one IP communication. 
Broadcast 
One-to-all IP communication (limited: 255.255.255.255; directed: subnet 
broadcast). 
Multicast 
One-to-group IP communication (range: 224.0.0.0 – 239.255.255.255). 
Private Address 
Non-routable IP ranges used internally: 10/8, 172.16/12, 192.168/16. 
NAT 
Network Address Translation — allows private hosts to communicate via a 
public IP. 
IPv6 
Next-gen IP with 128-bit addresses, designed to replace IPv4.
